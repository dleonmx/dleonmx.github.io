Arcade Monitoring System

What it is

Designed and built a fully functional arcade machine from scratch with the goal of operating it as a real-world business. The system combines custom hardware, embedded systems, and backend monitoring to manage the machine remotely and collect operational metrics. The arcade cabinet was designed, assembled, and deployed in a physical location where it currently operates as a revenue-generating machine.

Video

How it works

The arcade cabinet was physically designed and assembled to operate reliably in a public environment. A Raspberry Pi runs a customized Debian-based system configured to boot directly into the arcade game interface, removing visible traces of the operating system to create a seamless arcade experience.

An Arduino Micro handles hardware inputs and coin validation. A custom program allows the joystick and buttons to interface with the system while also integrating a coin sorter that enables credit-based gameplay.

When the machine powers on, a Python script automatically runs in the background and sends periodic POST requests to a Flask server. This backend collects operational data such as machine temperature, uptime, total revenue collected, and usage timestamps.

The data is stored and displayed through a web dashboard that allows remote monitoring of the machine's performance and activity. This enables real-time insights into machine health, usage patterns, and revenue metrics.

Tech used

  • Raspberry Pi
  • Arduino Micro
  • Python
  • Flask
  • Debian (custom boot configuration)
  • Embedded Systems Integration
  • Hardware Prototyping
